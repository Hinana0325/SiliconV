# devices — 设备模拟

客户可见的模拟硬件设备。所有设备均为内存映射（MMIO）并采用中断驱动。

## 设备列表

| 设备 | MMIO 基地址 | IRQ | 状态 |
|------|-------------|-----|------|
| [PL011 UART](uart/) | `0x10000000` | 32 | ✅ |
| [Virtio-MMIO 传输层](transport/) | `0x10001000`+ | 33+ | ✅ |
| [Virtio-Blk](virtio-blk/) | 通过传输层 | 通过传输层 | ✅ |
| [Virtio-Net](virtio-net/) | 通过传输层 | 通过传输层 | 🔲 |
| [Virtio-GPU](virtio-gpu/) | 通过传输层 | 通过传输层 | 🔲 |
| [Virtio-Input](virtio-input/) | 通过传输层 | 通过传输层 | 🔲 |
| [Virtio-Console](virtio-console/) | 通过传输层 | 通过传输层 | 🔲 |

## 架构

```
客户内核
    │
    ▼ (MMIO 读/写)
┌──────────────────────────────┐
│  Virtio-MMIO 传输层           │  ← transport/
│  ┌──────┬──────┬──────┬───┐ │
│  │ BLK  │ NET  │ GPU  │...│ │  ← 各个设备
│  └──────┴──────┴──────┴───┘ │
├──────────────────────────────┤
│  PL011 UART                  │  ← uart/
└──────────────────────────────┘
    │
    ▼ (IRQ 注入)
  GICv3 (core/irq/)
```

## 设计原则

- **Virtio 1.2 兼容** — 标准驱动开箱即用
- **MMIO 传输** — 无 PCI 复杂性；更简单且足以满足 ARM64
- **最小可行实现** — 只实现客户实际使用的功能
- **MMIO 退出驱动** — 设备逻辑在客户访问设备寄存器时执行
