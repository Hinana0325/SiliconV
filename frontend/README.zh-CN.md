# frontend — 用户界面

SiliconV 的前端应用。多个 UI 选项适用于不同平台。

## 前端列表

| 前端 | 平台 | 状态 | 描述 |
|------|------|------|------|
| [cli/](cli/) | 全平台 | ✅ | 命令行界面（主要） |
| [cocoa/](cocoa/) | macOS | 🔄 | 原生 macOS 应用（Cocoa/ObjC） |
| [qt/](qt/) | 全平台 | 🔲 | 跨平台 Qt GUI |
| [web/](web/) | 全平台 | 🔲 | 基于 Web 的控制面板 |

## 架构

```
┌──────────────────────────────────┐
│         前端层                    │
│  ┌──────┬───────┬─────┬──────┐  │
│  │ CLI  │ Cocoa │ Qt  │ Web  │  │
│  └──┬───┴───┬───┴──┬──┴──┬───┘  │
├─────┴───────┴──────┴─────┴──────┤
│         SiliconV 核心            │
│  （VM 管理器、设备、GPU）         │
└──────────────────────────────────┘
```

## CLI 前端

主要界面。从命令行启动 SiliconV：

```bash
./build/sv-cli -k Image -r rootfs.img --cpus 4 --memory 4096
```

## Cocoa 前端（macOS）

原生 macOS 应用，包含：
- 带显示输出的 VM 窗口
- 菜单栏控制
- 串口控制台查看器
- VM 生命周期管理

## 添加新前端

1. 在 `frontend/` 下创建目录
2. 链接 `libsiliconv`（核心库）
3. 实现 VM 生命周期回调（启动、停止、暂停）
4. 将 virtio-gpu 的显示输出路由到你的 UI
5. 在 `CMakeLists.txt` 中添加构建目标
